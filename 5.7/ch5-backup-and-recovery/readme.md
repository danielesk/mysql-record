# 前言

备份数据库很重要，这样您就可以恢复数据，并在出现问题（如系统崩溃、硬件故障或用户错误删除数据）时重新启动和运行。在升级MySQL安装之前，备份也是必不可少的保护措施，它们可用于将MySQL安装转移到其他系统或设置复制从属服务器。

MySQL提供了多种备份策略，您可以从中选择最适合安装要求的方法。本章讨论了您应该熟悉的几个备份和恢复主题：

- 备份类型：逻辑备份与物理备份、完全备份与增量备份等等。

- 创建备份的方法。

- 恢复方法，包括时间点恢复。

- 备份计划、压缩和加密。

- 表维护，以恢复损坏的表。
